# 训练减少显存占用完整方案（通用：NLP / CV / 时空 ConvLSTM 全部适用）
## 先明确最重要结论：
✅ 显存 ≠ 计算量 (FLOPs)
很多显存优化不会让训练变快，甚至变慢，只是少占显存，可以开更大 batch、更大模型。<br>
显存四大占用来源：
1. 模型参数权重
2. 前向传播每一层中间激活值（大头！NLP、时序模型最容易爆这里）
3. 梯度
4. 优化器状态（Adam 占显存非常多）
## 一、零改网络结构，直接生效（优先试，通用所有模型：Transformer、LSTM、CNN‑3D）
1. 混合精度训练（AMP）<br>
原理：前向用 FP16/BF16 计算，参数、梯度保存 FP32。<br>
激活、中间特征显存占用直接减半。<br>
NLP：BERT、Transformer 必开<br>
ConvLSTM 流场预测：BF16 更稳定，不容易梯度下溢<br>
注意：CPU 不支持 FP16，只有 GPU 生效，CPU 下开 amp 会直接报错类型异常<br>
```python
#新版写法
scaler = torch.amp.GradScaler("cuda")
with torch.amp.autocast(device_type="cuda", dtype=torch.bfloat16):
    loss = model(x)
```
缺点：不降低计算量，速度小幅提升；Windows 部分显卡对 BF16 支持一般。
2. 梯度检查点 gradient checkpointing
核心：牺牲少量计算时间，删掉前向中间激活，反向时重新前向计算一遍得到梯度<br>
👉大幅砍掉【中间激活】占用的显存（NLP 超长序列、ConvLSTM 时序堆叠显存爆炸首选）<br>
NLP 用法：transformer 每层开启 checkpoint，超长文本 4k/8k 序列防 OOM<br>
ConvLSTM 用法：对每个时序 Cell 内部开启 checkpoint，不要在外层循环套<br>
重要参数：use_reentrant=False，新版 pytorch 必须关闭，避免时序梯度 bug<br>
```python
from torch.utils.checkpoint import checkpoint
#在前向里调用
h_next = checkpoint(self.conv_op, x, h_prev, use_reentrant=False)
```
缺点：训练速度慢 20%‑35%，重复运算换取显存。
3. 梯度累加（不减少单步显存，等效扩大 batch）
不用开大 batch，多次迭代再更新一次参数
```python
accum_step=4
loss = loss / accum_step
loss.backward()
if (i+1)%accum_step==0:
    optimizer.step()
    optimizer.zero_grad()
```
⚠不会降低显存，只是不用一次性塞超大 batch 进显存。
## 二、优化器层面（显存收益巨大）
1. AdamW → 8bit‑Adam（bitsandbytes 库）
Adam 正常需要保存 2 份动量参数，显存开销≈3 倍参数量<br>
8bit 优化器把优化器状态压缩到 8bit，优化器显存直接降低 75%<br>
适用：NLP 大模型微调，时空模型同样能用，几乎不损失精度<br>
2. 改用 SGD，不用 Adam<br>
不需要维护动量，优化器显存极低，代价收敛更慢
## 三、网络输入 / 超参调优（同时降显存 + 降计算量）
这类方案既省显存，又加快训练，属于双重收益
1. 减小 batch_size（最直接，缺点迭代变多）
2. NLP：减小最大序列长度 seq_len；你的项目：减小时间步 T、降低图像分辨率 H,W
3. NLP：减小 hidden_dim；你的项目：降低 base_channel 通道数
4. NLP：减少 transformer 层数 / 头数；你的项目：减少 ConvLSTM 堆叠层数
## 四、模型层面改造（改代码结构，通用）
1. 激活重写 / 激活分片<br>
超长序列 NLP：不要一次性把完整序列送入 transformer，分块计算<br>
时序 ConvLSTM：逐时间步循环，不要一次性送入全部 T 帧做并行计算<br>
2. 选择性冻结部分层权重<br>
冻结层不保存梯度，梯度显存直接消失<br>
适合迁移学习，预训练微调场景，从零训练不推荐。<br>
3. 重写层，使用更小中间特征<br>
NLP：不用大中间维度 FFN<br>
CV / 时空：bottleneck 瓶颈结构，中间通道压缩<br>
## 五、高级分布式 / 显存卸载方案（超大模型场景）
1. 梯度 / 参数 CPU 卸载（offload）<br>
把不活跃的参数、优化器状态放到 CPU 内存，计算时再加载到 GPU<br>
DeepSpeed、Accelerate 库封装好了，NLP 大模型微调最常用，小项目一般不用。<br>
2. 模型并行（张量并行）<br>
一层放显卡 1，一层放显卡 2，拆分单卡显存压力，需要多 GPU。<br>
## 📌【分场景快速选型对照表】
|手段|NLP(Transformer)|ConvLSTM时空模型|是否减少 FLOPs (运算量)|
| ---- | ---- | ---- | ---- |
|BF16 混合精度|✅强烈推荐|✅强烈推荐|❌否|
|Gradient Checkpoint|✅长文本必开|✅多层时序必开|❌否（反而略增）|
|8bit 优化器|✅大模型微调|⭕可选|❌否|
|降低序列长度 / T 帧长|✅首选|✅首选|✅是|
|减小隐藏维度 / 通道|✅|✅|✅是|
## 重点区分一句话总结（避免混淆）
只降显存，计算量不变甚至增加：混合精度、梯度检查点、8bit 优化器<br>
既降显存，又减少运算量、加快训练：减序列长度、降分辨率、减通道、减少层数
